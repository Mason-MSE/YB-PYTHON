# Clinic Service System - Use Case Descriptions

| Use Case Name | Primary Actor | Description | Pre-conditions | Basic Flow | Extensions/Includes |
|---------------|---------------|-------------|----------------|------------|---------------------|
| **Check doctor available** | Patient | Patient checks the availability of doctors for appointments | 1. Patient is registered in the system<br>2. Patient is logged in | 1. Patient selects "Check Availability"<br>2. System shows list of doctors<br>3. Patient selects a doctor<br>4. System displays available time slots | **Included by**: Request an appointment |
| **Request an appointment** | Patient | Patient books an appointment with a doctor | 1. Patient is logged in<br>2. Doctor has available slots | 1. Patient selects "Book Appointment"<br>2. System shows available doctors/times<br>3. Patient selects preferred slot<br>4. System confirms appointment<br>5. System sends confirmation | **Includes**: Check doctor available |
| **Cancel an appointment** | Patient | Patient cancels a previously booked appointment | 1. Patient has an existing appointment<br>2. Patient is logged in | 1. Patient views appointments<br>2. Patient selects appointment to cancel<br>3. System asks for confirmation<br>4. Patient confirms cancellation<br>5. System updates doctor's schedule | None |
| **Patient record** | Patient, Doctor | View and manage patient's medical records | 1. User is authenticated<br>2. Patient record exists | 1. User accesses patient records<br>2. System displays record information<br>3. (For Doctor) Update/add information<br>4. System saves changes | None |
| **Prescribe Medication** | Doctor | Doctor creates a prescription for a patient | 1. Doctor is logged in<br>2. Patient is registered<br>3. Consultation completed | 1. Doctor accesses patient record<br>2. Doctor adds medication details<br>3. System validates prescription<br>4. System sends to pharmacist | **Includes**: Make payment |
| **Make payment** | Patient | Patient makes payment for services | 1. Service is provided (consultation, medication)<br>2. Bill is generated | 1. System generates invoice<br>2. Patient selects payment method<br>3. System processes payment<br>4. System confirms payment | **Extends to**: Process payment (via Payment Gate) |
| **Process Prescription** | Pharmacist | Pharmacist processes and fulfills a prescription | 1. Prescription exists<br>2. Pharmacist is logged in | 1. Pharmacist receives prescription<br>2. Pharmacist prepares medication<br>3. Pharmacist updates prescription status<br>4. System notifies patient | None |
| **Process payment** | Payment Gate | External system processes financial transaction | 1. Payment request received<br>2. Connection to payment network | 1. Receives payment details<br>2. Validates transaction<br>3. Processes with bank<br>4. Returns success/failure status | **Extension of**: Make payment |
